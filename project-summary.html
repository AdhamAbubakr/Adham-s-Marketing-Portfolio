01 · Executive Summary

Where We Are

In one sprint, we transformed a static portfolio into the foundation of an agency-operations platform with multi-tenant security, real client onboarding, and the ability to manage every brand from one dashboard.

✅ MVP delivered in days, not weeks. The site is live, the database is structured, clients can sign up and access their own portal, and you have an admin command-center that scales as you scale.

The big picture

We built a production-grade Phase 1 + Phase 2 SaaS prototype covering:

  • Public Portfolio — full brand identity, services, case studies, lead capture
  • Admin Dashboard — leads, clients, per-client metrics, tracking pixels
  • Client Portal — signup, login, service-based dashboards, real-time data
  • Multi-tenant Security — role-based access; clients only see their own numbers
  • Brand System — logo files, palette, full PDF brand guide

What this means for the business

You can already start onboarding clients today. They get a real login, a private dashboard, and live numbers — the same experience an agency 10× our size would offer. Scope for monthly services becomes easier to pitch, because the “you’ll get a portal” line is now a real deliverable.

⚠️ Reality check. The free tier (Netlify + Supabase) is fine for piloting with 1–3 clients. Beyond that, the migration plan in this doc moves us to Hostinger + Supabase Pro for ~$30–40/month total.

04 · Architecture

Stack & How It Fits Together

Tech stack

  • Frontend — single-file HTML / CSS / vanilla JS (no framework, no build step)
  • Hosting — Netlify (free tier; auto-deploys from GitHub on push)
  • Source control — GitHub (github.com/AdhamAbubakr/Adham-s-Marketing-Portfolio)
  • Database — Supabase Postgres (free tier; project rgofucjhmiiyygbcnoim)
  • Auth — Supabase Auth (email + password; verification on)
  • Charts — Chart.js (CDN)
  • Email notifications — Web3Forms (free, 250/mo)
  • Type — Google Fonts (Space Grotesk + Inter)

Request flow — public visitor

Visitor opens adhamabobakr.netlify.app
↓
Static HTML loads from Netlify CDN
↓
tracking.js fetches pixel IDs from Supabase → injects Meta/GA4/etc.
↓
Visitor fills form → supabase.from('leads').insert()
↓
Web3Forms POST → email lands at adhambakrsalah@gmail.com

Request flow — client login

Client opens /signup.html → signs up
↓
supabase.auth.signUp() → trigger creates user_role = 'client'
↓
Client record auto-created OR linked by email
↓
Login → role check → redirect to /portal.html
↓
Portal fetches client/campaigns/metrics filtered by RLS

Request flow — admin

Admin opens /admin.html → signs in
↓
role check → is_admin() returns true
↓
Full access to leads / clients / campaigns / metrics / pixels
↓
Realtime subscription on new leads → auto-update dashboard

Database tables (Supabase)

TablePurposeKey Columns
leadsPublic form submissionsid, name, email, phone, service, budget, message, source, status
site_settingsPixel IDs & custom codekey, value, updated_at
clientsBrand profiles + auth linkid, auth_user_id, name, slug, services, platforms, monthly_budget, contact_*
client_campaignsPer-client campaignsid, client_id, name, platform, objective, budget, external_url, status

client_activityAction logid, client_id, type, title, description
user_rolesAdmin vs clientuser_id, role

Security model (RLS policies)

  • leads — anyone INSERTs; only authenticated read/update/delete
  • site_settings — anyone reads (public site needs pixel IDs); admin writes
  • clients / campaigns / metrics / activity — admin: full access; client: read-only on own row(s)
  • user_roles — user reads their own role only
  • Trigger — new auth signup auto-assigned role='client'
  • Function — is_admin() returns true only for role='admin'
06 · Workflows

How You Use the System

Onboarding a new client

  1. Discovery call → agree on scope
  2. Open /admin.html → Clients tab → + Add New Client
  3. Fill name, contact email, country, platforms, monthly budget
  4. Send the client the link adhamabobakr.netlify.app/signup.html
  5. Client signs up with the same email → system auto-links to your record
  6. Client picks services → their dashboard goes live

Weekly cycle (per client)

  1. Run campaigns on Meta / Google / TikTok as usual
  2. Open the client’s page in /admin.html
  3. Click + Log Numbers → enter spend, reach, conversions, revenue per platform
  4. Charts update instantly; client portal reflects the new data on next login
  5. Add/update campaigns; paste their platform URL so you both can jump back to Meta in one click

Monthly reporting (today)

  1. Open client dashboard at end of month
  2. Screenshot the stat cards + charts
  3. Send to the client via WhatsApp/email + a brief commentary

Note: automated PDF export is on the roadmap — see Section 8.

Lead management

  • New lead arrives → you get an email AND it shows in Leads tab
  • Search by name/email/phone, filter by status
  • Change status (new → contacted → converted → lost) — single click
  • Export CSV anytime for cold reach-outs / CRM imports

Pixel deployment

  • Want to track website traffic with Meta? Paste Pixel ID in Pixels tab → Save
  • Pixel goes live on every page within seconds — no redeploy
  • Removing the ID disables the pixel instantly
07 · Migration Plan

From Free Tier → Production Infra

When the system gets serious traction (3+ active clients), here’s the upgrade path.

Recommended stack

🏠 Hosting

$3–10/mo

Hostinger Premium — static hosting, free SSL, custom domain, fast CDN. Or stay on Netlify Pro ($19) if simplicity matters more than savings.

🗄️ Database

$25/mo

Supabase Pro — 8 GB DB, 250 GB bandwidth, 100 GB storage, daily backups, support. Same project, just upgrade.

🌐 Domain

$10–15/yr

Custom domain like adhammarketing.com. Hostinger or Namecheap. Add as alias in Netlify or point to Hostinger.

Total cost: ~$30–40/month + $10–15/year — affordable once you have 3+ paying clients on retainer.

Migration checklist

  1. Buy domain at Hostinger or Namecheap (5 minutes)
  2. Upgrade Supabase to Pro plan (1 click, no migration needed — same project keeps running)
  3. Connect domain to Netlify: Site Settings → Domain Management → Add custom domain → follow DNS instructions
  4. OR move to Hostinger: upload portfolio/ folder via File Manager or FTP → point domain
  5. Update Supabase Site URL in Authentication settings to your new domain (otherwise email verification breaks)
  6. Test signup flow end-to-end on the new domain
  7. Set up daily backups in Supabase dashboard (Settings → Database → Backups)

Disaster recovery

If anything ever goes wrong (Netlify down, Supabase data lost, account deleted), you can rebuild from this PDF + the GitHub repo + the database-backup.sql file in under one day:

  1. Clone the GitHub repo to any host (Netlify / Vercel / Hostinger)
  2. Create a new Supabase project
  3. Run database-backup.sql in the SQL editor
  4. Update supabase-config.js with the new project URL + anon key
  5. Push to host — you’re back online

09 · Backup

If Anything Breaks

Three artifacts make this project recoverable from total loss.

📄 This PDF

Full architectural blueprint. Even with no code, an experienced dev can rebuild everything from this document in 2–3 days.

🗄️ database-backup.sql

One file recreates every table, every policy, every trigger, every helper function. Run on a fresh Supabase project → instant database.

🐙 GitHub Repo

github.com/AdhamAbubakr/Adham-s-Marketing-Portfolio — full git history, every commit, every revert path. The repo IS the source of truth.

🤖 AI-CONTEXT.md

Hand this to any new AI session (Claude, ChatGPT, Gemini) and it can pick up exactly where this work left off.

If Netlify goes down

  1. Sign in to Vercel / Cloudflare Pages with your GitHub
  2. Import the repo — auto-deploys in 30 seconds
  3. Update DNS to point to the new host

If Supabase data is corrupted

  1. Restore from latest daily backup (Supabase Pro — included)
  2. Or: create new project → run database-backup.sql → restore from any CSV exports you kept

If you lose access to your account

  1. Recover GitHub via 2FA backup codes
  2. Recover Netlify / Supabase via email reset
  3. Worst case — rebuild from this PDF on a new host (one full day of work)
Your business survives any single failure. No single point of failure can take down your data, your code, AND your knowledge of the system at the same time. That’s why we built it this way.

What’s Next

  2. Build Phase 3 now. Brief form + strategy module + monthly PDF export. Brings the system to ~55%.
